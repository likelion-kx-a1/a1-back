# 테스트 및 시뮬레이션 가이드 (Mock ↔ 실제 API 전환)

`generation` 도메인이 이미지/영상을 만드는 3개 외부 API(Claude, GPT Vision, fal.ai)를 **비용 없이 Mock으로 테스트**하는 방법과, 필요할 때만 **최소 비용으로 실제 API**를 검증하는 방법을 정리합니다.

---

## 1. 전환 메커니즘 — Spring `@Profile`

3개 Port(`PromptGenerationPort`, `ImageAnalysisPort`, `FalGenerationPort`) 각각에 어댑터가 2개씩 있고, 활성 프로필로 어떤 Bean이 뜰지 결정됩니다.

| Port | Mock 어댑터 (`@Profile("local")`) | 실제 어댑터 (`@Profile("!local")`) |
|---|---|---|
| `PromptGenerationPort` | `claude/MockPromptGenerationAdapter` | `claude/ClaudePromptGenerationAdapter` (Anthropic Messages API) |
| `ImageAnalysisPort` | `openai/MockImageAnalysisAdapter` | `openai/GptVisionImageAnalysisAdapter` (OpenAI Chat Completions) |
| `FalGenerationPort` | `fal/MockFalGenerationAdapter` | `fal/FalGenerationAdapter` (fal.ai 큐 API) |

`src/main/resources/application.yml`에 `spring.profiles.default: local`로 되어 있어서, **아무 설정도 안 하면 `local` 프로필이 켜지고 3개 전부 Mock으로 동작합니다.**

```bash
# Mock (기본값, 아무 설정 없이 실행)
./gradlew bootRun

# 실제 API로 전환 — local이 아니기만 하면 되지만 관례상 prod 사용
SPRING_PROFILES_ACTIVE=prod ./gradlew bootRun
```

한쪽만 골라 전환하는 게 아니라 3개가 한 세트로 같이 전환됩니다(개별 Port마다 프로필을 다르게 줄 수는 없습니다). Mock/실제를 섞어 쓰려면 `@Profile` 조건을 분리하거나 `@ConditionalOnProperty`로 바꿔야 하는데, 현재는 그런 요구가 없어 하지 않았습니다.

## 2. Mock 모드 — 로컬 개발 기본값 (비용 $0)

`local` 프로필에서 각 어댑터가 실제로 동작하는 방식:

- **`MockPromptGenerationAdapter`** — instruction 앞에 `[MOCK-REFINED]`를 붙이고 `anti-gravity physics, cinematic lighting...` 문구를 덧붙여 즉시 반환.
- **`MockImageAnalysisAdapter`** — `[MOCK-ANALYSIS]` + 더미 분석 텍스트를 즉시 반환.
- **`MockFalGenerationAdapter`** — `submit()` 시 `mock-<UUID>` requestId를 즉시 발급하고 상태를 `IN_QUEUE`로 둡니다. `poll()`은 **같은 requestId로 2번째 호출부터** `COMPLETED` + 더미 미디어 URL(`mock-image.png` / `mock-video.mp4`, modelCode에 `video` 포함 여부로 판별)을 반환합니다(`pollCounts`라는 `ConcurrentHashMap`으로 호출 횟수를 셉니다).
- **`GenerationVideoPollingScheduler`**가 5초마다 `QUEUED`/`PROCESSING` 상태인 job을 스캔해서 자동으로 poll → `COMPLETED`면 `MediaStoragePort`(Mock 환경에서는 Mock 스토리지 어댑터)로 "업로드"해 `s3Url`을 채웁니다. 클라이언트가 수동으로 상태 조회를 안 해도 5~15초 뒤에는 `COMPLETED` + `s3Url`이 채워져 있습니다.

### 준비물

1. `docker compose up -d` — Postgres(5433)/Redis(6379)/MinIO(9000). AI API 쪽은 전혀 필요 없습니다(키가 없어도, 인터넷이 없어도 동작).
2. `./gradlew bootRun` (프로필 지정 없이 — 기본값이 `local`).
3. `users`/`chats`에 FK를 만족하는 테스트 row 최소 1개씩 필요 (`로컬_개발환경_설정_가이드.md` 3장 참고).

## 3. 실제 API 모드 — 진짜 Claude / GPT Vision / fal.ai 호출

### 3.1 필요한 환경변수

`ANTHROPIC_API_KEY`, `CLAUDE_MODEL`, `ANTHROPIC_VERSION`, `OPENAI_API_KEY`, `OPENAI_VISION_MODEL`, `FAL_API_KEY` — 상세는 `로컬_개발환경_설정_가이드.md` 4장 표를 참고하세요.

`RestClient` Bean 3개는 `global/config/AiClientConfig.java`에 있고, 인증 헤더가 각각 다릅니다:
- fal.ai: `Authorization: Key <FAL_API_KEY>`
- Claude: `x-api-key: <ANTHROPIC_API_KEY>` + `anthropic-version` 헤더
- OpenAI: `Authorization: Bearer <OPENAI_API_KEY>`

### 3.2 ⚠️ 실제 API 모드 전환 전 반드시 확인할 것

`application.yml`/`.env.example`에 실제 키가 평문으로 커밋되어 있는 미해결 보안 이슈가 있습니다. 실제 API로 전환하기 **전에** `트러블슈팅_가이드.md` 5장을 반드시 먼저 읽으세요.

### 3.3 실행

```bash
docker compose up -d          # Postgres/Redis/MinIO
SPRING_PROFILES_ACTIVE=prod ./gradlew bootRun
```

`prod` 프로필이어도 `DB_URL`/`REDIS_HOST` 등은 `.env` 값을 그대로 쓰므로, **DB만 로컬 Postgres를 계속 보게 하면서 AI 어댑터만 실제로 바꿔 검증**하는 것이 가능합니다(운영 DB에 연결하려는 게 아니라면 `.env`의 `DB_URL`을 `localhost:5433`으로 둘 것).

### 3.4 실패 시 응답 형식

키가 비어있거나 잘못됐거나 공급자가 4xx/5xx를 반환하면 `RestClientResponseException`이 `GlobalExceptionHandler`에서 `PROVIDER-002`(502, `AI_PROVIDER_REQUEST_FAILED`)로 매핑되어 프로젝트 표준 `ErrorResponse` 포맷으로 내려옵니다. 이때도 `generation_jobs` row는 `FAILED` 상태 + `errorMessage`로 저장되므로 DB에서 실패 사유를 그대로 확인할 수 있습니다.

## 4. 비용 통제 전략 (실제 API를 검증해야 할 때)

실제 API는 "회귀 테스트"가 아니라 **"1회성 스키마 확인"** 용도로만 사용합니다 — 매 커밋/PR마다 실제 API를 호출하지 않습니다. 전체 개발 공정의 95% 이상은 2~3장의 Mock으로 완료해야 합니다.

- **가입 무료 크레딧**: fal.ai는 회원가입(GitHub 연동 권장) 시 기본 $1~2 내외의 무료 크레딧을 제공합니다. 가벼운 통합 검증용으로 충분합니다.
- **저비용 모델/옵션 선택**: 영상 모델은 초당 과금이라 비쌉니다.
  - 피해야 할 모델: Kling Pro/4K급 — 초당 약 $0.42, 5초만 뽑아도 $2.1로 무료 크레딧이 바로 소진됩니다.
  - 권장 모델: Kling Turbo/Standard급 — 초당 약 $0.11 수준. 해상도를 가장 낮게, 길이는 기본값(5초)으로 설정하면 1회 검증에 약 $0.5(≈700원) 내외로 끝납니다.
  - 응답 스키마 확인용 1회성 cURL 예시:
    ```bash
    curl -X POST "https://queue.fal.run/fal-ai/kling-video/v3/turbo/standard/text-to-video" \
      -H "Authorization: Key YOUR_FAL_KEY" \
      -H "Content-Type: application/json" \
      -d '{"prompt": "A close-up of a weightless sphere of water floating in neon light, slow-motion, highly detailed", "duration": 5, "aspect_ratio": "16:9"}'
    ```
    이 요청의 응답 `request_id`와 JSON 구조를 캡처해 Mock 데이터/스키마 설계에 반영합니다.
- **fal.ai 대시보드 하드 리밋**: 실수로 무한 루프가 발생해 API가 계속 호출되더라도 결제가 차단되도록, fal.ai Billing 대시보드에서 Auto-Refill을 끄고 Spend Limit(일/월 한도)을 최소 금액(예: $5~10)으로 걸어둡니다.

## 5. Postman/curl 요청 예시

로컬 실행 후(`./gradlew bootRun`, 포트 8080 기준) 아래 요청을 그대로 보내면 됩니다. 실제 응답을 받으려면 `FAL_API_KEY`/`ANTHROPIC_API_KEY`/`OPENAI_API_KEY`가 `.env`에 설정돼 있어야 합니다(Mock 모드에서는 키 없이도 200이 옵니다).

fal.ai 공식 페이지 기준 확인된 모델 슬러그(2026-07-07):
- `openai/gpt-image-2` / `openai/gpt-image-2/edit` — 이미지 생성/편집
- `fal-ai/kling-video/o3/pro/text-to-video` — 텍스트→영상
- `bytedance/seedance-2.0/mini/image-to-video` — 이미지→영상

### 1. gpt-image-2 (텍스트 → 이미지)

```
POST /api/v1/generation/fal-jobs
Content-Type: application/json

{
  "userId": 1,
  "chatId": 1,
  "jobType": "IMAGE_GENERATION",
  "modelCode": "openai/gpt-image-2",
  "input": {
    "prompt": "a cinematic product shot of a coffee cup on a wooden table",
    "image_size": "landscape_4_3",
    "quality": "high",
    "num_images": 1
  }
}
```

### 2. kling o3 pro (텍스트 → 영상)

```
POST /api/v1/generation/fal-jobs
Content-Type: application/json

{
  "userId": 1,
  "chatId": 1,
  "jobType": "VIDEO_GENERATION",
  "modelCode": "fal-ai/kling-video/o3/pro/text-to-video",
  "input": {
    "prompt": "drone shot flying over a mountain range at sunrise",
    "duration": "5",
    "aspect_ratio": "16:9"
  }
}
```

### 3. seedance 2.0 mini (이미지 → 영상)

```
POST /api/v1/generation/fal-jobs
Content-Type: application/json

{
  "userId": 1,
  "chatId": 1,
  "jobType": "VIDEO_GENERATION",
  "modelCode": "bytedance/seedance-2.0/mini/image-to-video",
  "input": {
    "prompt": "the camera slowly zooms in as the subject smiles",
    "image_url": "https://example.com/source-image.jpg",
    "resolution": "720p"
  }
}
```

`jobType`은 `GenerationType` enum 값 중 하나여야 합니다(`IMAGE_GENERATION`, `VIDEO_GENERATION`, `REVERSE_PROMPT`, `IMAGE_VARIATION`, `PROMPT_REGENERATION`). 다른 값이면 `COMMON-001`(400)로 거부됩니다.

### 상태 조회 (공통)

```
GET /api/v1/generation/fal-jobs/{jobId}/status
```

submit 응답의 `id`가 jobId입니다. modelCode/externalRequestId는 저장된 job row에서 서버가 알아서 꺼내 fal.ai에 다시 poll을 날립니다.

### Claude / GPT Vision

```
POST /api/v1/generation/prompts        (Claude Sonnet - 사진 → 프롬프트 생성)
POST /api/v1/generation/reverse-prompts (GPT Vision - 역프롬프팅)

{
  "userId": 1,
  "chatId": 1,
  "imageBase64": "<base64 인코딩된 이미지>",
  "mimeType": "image/png",
  "instruction": "이 사진을 보고 영상 생성용 프롬프트를 만들어줘"
}
```

전체 요청/응답 필드 정의는 `API_명세서.md`를 참고하세요.

### 결론

Mock 기반 전체 파이프라인(프롬프트 보정 → 역프롬프트 → fal 제출 → 5초 자동 폴링 → S3 업로드 시뮬레이션 → 완료)과 주요 에러 케이스가 현재 코드 기준으로 전부 정상 동작합니다. 실제 API(`!local` 프로필) 전환 자체도 코드 레벨(`@Profile` 분기, `RestClientConfig`, 헤더 구성)로는 문제가 없어 보이나, **`트러블슈팅_가이드.md` 5장의 키 유출 문제를 해결하기 전까지는 실제 키로 전환해 호출하는 것을 권장하지 않습니다.**
